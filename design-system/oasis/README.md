# @lyel/oasis

**Oasis** design system: JSON tokens with SCSS (web) and TS (mobile) exports, including responsive breakpoints.

## Target structure

```
packages/design-system/oasis/
├── package.json                 # @lyel/oasis, exports . + ./scss + ./tokens
├── README.md
├── tokens/                      # Source of truth (Design Tokens JSON)
│   ├── base/
│   │   ├── breakpoints.json     # xs, sm, md, lg, xl, xxl (responsive)
│   │   ├── colors.json
│   │   ├── spacing.json
│   │   ├── typography.json      # fontFamily, fontSize, fontWeight
│   │   ├── radius.json
│   │   └── shadows.json
│   └── index.json
├── build/
│   └── run.mjs                  # Reads tokens/base/*.json → generates dist/
├── dist/                        # Generated by npm run build
│   ├── scss/
│   │   ├── _variables.scss      # Variables $oasis-*
│   │   ├── _breakpoints.scss    # Map + mixins oasis-breakpoint-up/down
│   │   └── oasis.scss           # Entry @forward variables + breakpoints
│   └── tokens/
│       └── index.ts             # breakpoints, color, spacing, typography, radius, shadow
└── src/                         # (Optional) Web components later
```

## Usage

- **Web (backoffice)**: import SCSS variables and mixins.
  ```scss
  @use "@lyel/oasis/scss";
  // Variables: $oasis-color-primary, $oasis-spacing-4, etc.
  // Responsive: @include oasis.oasis-breakpoint-up(md) { ... }
  ```
- **Mobile**: import TS tokens.
  ```ts
  import { breakpoints, color, spacing } from "@lyel/oasis";
  ```

## Build

```bash
cd packages/design-system/oasis && npm run build
```

Generates `dist/scss/` and `dist/tokens/` from `tokens/base/*.json`.
